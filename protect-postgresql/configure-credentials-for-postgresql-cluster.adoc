= 設定 PostgreSQL 叢集的認證資料
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
您必須設定 PostgreSQL 叢集的叢集使用者認證資料才能執行資料保護作業。

.開始之前
* 如果 PostgreSQL 叢集未配置有效的認證資料，則僅自動探索的資源會顯示紅色掛鎖圖示。如果在後續的探索作業中，發現配置的認證資料不正確或無法提供對叢集本身的存取權限，則紅色掛鎖圖示將再次出現。
* 您必須配置有效的 PostgreSQL 叢集認證資料才能保護資源或將其新增至資源群組以執行資料保護作業。
* 憑證中配置的 PostgreSQL 使用者必須擁有足夠的權限才能存取叢集並執行備份和還原作業。該使用者至少應具有 *SUPERUSER* 角色或等效權限（例如 *pg_read_all_data*、*pg_write_all_data* 和 *replication* 權限）。
* SnapCenter 使用已設定的認證資料透過 *psql* 或 *libpq* 用戶端連線至 PostgreSQL 執行個體。請確保外掛程式主機上已安裝 PostgreSQL 用戶端工具。



NOTE: 在配置叢集之前，您必須先將 PostgreSQL 認證資料新增至 SnapCenter。

.步驟
. 在左側導覽窗格中，選擇 *Resources* ，然後從清單中選擇適用於 PostgreSQL 叢集的 SnapCenter 外掛程式。
. 在「資源」頁面中、從*檢視*清單中選取資源類型。
. （選用）選取 image:../media/filter_icon.gif["篩選圖示"] 並選取主機名稱。
+
然後您可以選擇 image:../media/filter_icon.gif["篩選圖示"] 關閉篩選窗格。

. 選取叢集，然後選取 *Configure credentials* 。
. 選擇已在 SnapCenter 中為 PostgreSQL 叢集使用者設定的認證資料。認證資料必須包含有效的使用者名稱和密碼，並具有執行資料保護作業所需的足夠權限。
. 選擇*確定*。


您可以從拓撲頁面修改叢集組態。
